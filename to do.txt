# ✅ 2. No CSRF Protection - Fixed with Flask-WTF

You accept many POST requests:

Examples:

```
/login
/register
/reset
/batch/assign
/generate/code
/export
```

But there is **no CSRF protection**.

Attack example:

A malicious site could silently send:

```
POST /batch/assign
```

while the user is logged in.

Fix with:

* Flask-WTF
* or token-based CSRF middleware

# ✅ 3. Password Stored in Session (Registration) - Fixed by hashing the password

You store the **plaintext password** inside the session:

```python
session["pending_registration"] = {
    ...
    "password": password,
}
```

Problems:

1️⃣ Sessions can be stolen
2️⃣ Password stored **unencrypted**
3️⃣ Bad security practice

Fix:

Store **hashed password** instead.

Example:

```python
from werkzeug.security import generate_password_hash

hashed = generate_password_hash(password)
```

Store:

```
"password_hash": hashed
```

# ✅ 4. Missing Rate Limiting - Fixed with Flask-Limiter

Sensitive endpoints:

```
/login
/reset/send-otp
/reset/verify-otp
/resend-otp-registration
```

Currently attackers can:

* brute force passwords
* brute force OTPs
* spam OTP emails

Fix using:

* Flask-Limiter

Example:

```
5 login attempts per minute
5 OTP attempts per minute
```

# ✅ 5. OTP Brute Force Vulnerability - Fixed in otpModel.py

OTP verification:

```python
verified, message = otp.verify_otp(username, otp_input)
```

There is **no attempt limit**.

Attack:

```
000000
000001
000002
...
999999
```

6-digit OTP can be brute forced quickly.

Fix:

Add:

```
max_attempts = 5
lockout_time = 5 minutes
```

# ✅ 6. Missing Secure Session Cookie Settings - Fixed in app.py

Your app does not configure session cookies securely.

Add:

```python
app.config.update(
    SESSION_COOKIE_SECURE=True,
    SESSION_COOKIE_HTTPONLY=True,
    SESSION_COOKIE_SAMESITE="Lax"
)
```

Prevents:

* XSS cookie theft
* CSRF amplification
